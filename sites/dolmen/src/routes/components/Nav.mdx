# Nav

The `Nav` component is used to display the primary navigation for the application,
usually in a sidebar. It contains links that allow users to move between different
views within the application.

    </Nav>
  </Card>
</DemoSection>

## Nav structure

The `Nav` component contains a list of items and group headings:

* `Nav.Group` represents the heading for a group of items.
* `Nav.Subroup` represents the heading for a nested subgroup of items.
* `Nav.Item` represents a navigation link.

## Client-side Navigation / Integration with Routers

By default, the `Nav.Item` component renders an anchor (`a`) element. However, you can pass
in a custom link component using the `as` property. This is useful for supporting routers
with client-side navigation. The link component must accept an `href` property and must
generally act like an anchor tag.

For example, if you are using [solid-router](https://github.com/solidjs/solid-router),
you can directly pass in the `A` component:

<SourceCode>
```tsx
import { A } from 'solid-router';

<Nav.Link as={A} href="./Nav">Installation</Nav.Link>
```
</SourceCode>

## Link States

The `disabled` property causes the link to be rendered as disabled.

The `current` property adds the `aria-current` attribute to the link, which causes it to be
displayed as selected. Note that many routers (such as solid-router) will automatically add the
`aria-current` attribute for you, so you don't need to bother with `current`.

## Next / Prev

The `Nav.Prev` and `Nav.Next` components can be used to navigate to the previous or next page:

<DemoSection>
  <Group>
    <Nav.Prev href="./Nav">
      <span>Site</span>
    </Nav.Prev>
    <Spacer />
    <Nav.Next href="./Nav">
      <span>Channels</span>
    </Nav.Next>
  </Group>
</DemoSection>
<SourceCode>
```tsx
  <Group>
    <Nav.Prev href="/">
      <span>Site</span>
    </Nav.Prev>
    <Spacer />
    <Nav.Next href="/">
      <span>Channels</span>
    </Nav.Next>
  </Group>
```
</SourceCode>

## Nav.Item properties

`Nav.Item` accepts all of the standard properties for an anchor element, including `href`.
Unlike a native anchor, the `href` is a required property.

In addition, it supports the following properties:
